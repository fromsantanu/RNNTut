## 🧩 Problem We Will Solve

Given a sequence:

```id="txt1"
"I love"
```

👉 Predict:

```id="txt2"
"AI"
```

---

## 🔑 Step 1: Convert Text to Numbers

Neural networks cannot understand text directly.

👉 So we convert words into numbers.

---

### 📦 Vocabulary Mapping

```python id="txtmap1"
vocab = ["I", "love", "AI", "you"]
word_to_idx = {word: i for i, word in enumerate(vocab)}

print(word_to_idx)
```

Output:

```id="txtmap2"
{'I': 0, 'love': 1, 'AI': 2, 'you': 3}
```

---

## 🧠 Why This Step?

👉 Model only understands numbers
👉 Each word becomes an index

---

## 🔄 Step 2: Create Training Data

```python id="txtdata1"
# Input sequences
X = [
    ["I", "love"],
    ["I", "love"],
    ["love", "you"]
]

# Targets
y = ["AI", "you", "AI"]

# Convert to numbers
X = [[word_to_idx[w] for w in seq] for seq in X]
y = [word_to_idx[w] for w in y]

import torch

X = torch.tensor(X)
y = torch.tensor(y)
```

---

## 🧠 Shape Understanding

```id="txtshape1"
X shape = (samples, sequence_length)
```

👉 Later we convert to proper RNN format

---

## 🔧 Step 3: Add Embedding Layer

Instead of raw numbers, we use **embeddings**:

👉 Convert each word into a meaningful vector

```python id="txtembed1"
embedding = nn.Embedding(num_embeddings=4, embedding_dim=5)
```

---

## 🧠 Intuition

* “I” → [0.2, 0.1, ...]
* “love” → [0.8, 0.5, ...]

👉 Similar words → similar vectors

---

## 🏗 Step 4: Build RNN Model

```python id="txtmodel1"
import torch.nn as nn

class TextRNN(nn.Module):
    def __init__(self):
        super(TextRNN, self).__init__()
        
        self.embedding = nn.Embedding(4, 5)
        self.rnn = nn.RNN(5, 8, batch_first=True)
        self.fc = nn.Linear(8, 4)
    
    def forward(self, x):
        x = self.embedding(x)
        
        out, _ = self.rnn(x)
        
        out = out[:, -1, :]
        out = self.fc(out)
        
        return out
```

---

## 🧠 What’s Happening

* Embedding → convert words to vectors
* RNN → process sequence
* Linear → predict next word

---

## ⚙️ Step 5: Train the Model

```python id="txttrain1"
model = TextRNN()
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=0.01)

for epoch in range(200):
    
    output = model(X)
    loss = criterion(output, y)
    
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    
    if epoch % 20 == 0:
        print(f"Epoch {epoch}, Loss: {loss.item():.4f}")
```

---

## 🔍 Step 6: Test Prediction

```python id="txttest1"
test_seq = torch.tensor([[word_to_idx["I"], word_to_idx["love"]]])

output = model(test_seq)

predicted_idx = torch.argmax(output, dim=1).item()

idx_to_word = {i: w for w, i in word_to_idx.items()}

print("Predicted word:", idx_to_word[predicted_idx])
```

## ⚠️ Limitations

* Works only on small vocabulary
* Cannot handle long context well (basic RNN)

👉 That’s why:

* LSTM / GRU / Transformers are used

---

## 📌 Key Takeaways

* Text must be converted to numbers
* Embedding gives meaningful representation
* RNN processes sequence
* Output predicts next word

---

## 📌 One-Line Summary

> RNN can learn patterns in text sequences and predict the next word using embeddings and sequential memory.
