# 📘 Chapter 8.1: Hyperparameter Tuning

---

## 🧠 Why This Chapter Matters

You’ve built RNN models that work.

But now the real question is:

> ❓ How do we make them **work better**?

👉 That’s where **hyperparameter tuning** comes in.

---

## 🔑 What is a Hyperparameter?

> A **hyperparameter** is a setting you choose **before training** the model.

👉 It controls **how the model learns**, not what it learns.

---

## 🧩 Simple Analogy

Think of cooking:

* Ingredients → data
* Recipe → model
* Cooking settings (flame, time, spice level) → hyperparameters

👉 Same dish, different settings → very different results

---

## 🔧 Important Hyperparameters in RNN

---

## 1️⃣ Learning Rate

```id="hp1"
lr = 0.01
```

### 🧠 What it controls:

* How fast the model learns

---

### ⚖️ Effect

| Value      | Result                |
| ---------- | --------------------- |
| Too high   | Model jumps, unstable |
| Too low    | Very slow learning    |
| Just right | Smooth learning       |

---

### 🧩 Analogy

Walking down a hill:

* Big steps → may fall
* Tiny steps → too slow

---

## 2️⃣ Hidden Size

```id="hp2"
hidden_size = 10
```

### 🧠 What it controls:

* Memory capacity of RNN

---

### ⚖️ Effect

| Value | Result                        |
| ----- | ----------------------------- |
| Small | Cannot learn complex patterns |
| Large | More powerful but slower      |

---

### 🧩 Analogy

Notebook size:

* Small notebook → limited memory
* Large notebook → more storage

---

## 3️⃣ Number of Layers

```id="hp3"
num_layers = 2
```

### 🧠 What it controls:

* Depth of model

---

### ⚖️ Effect

| Value           | Result           |
| --------------- | ---------------- |
| 1 layer         | Simple patterns  |
| Multiple layers | Complex patterns |

---

### 🧩 Analogy

Single teacher vs multiple teachers explaining deeply

---

## 4️⃣ Sequence Length

```id="hp4"
sequence_length = 5
```

### 🧠 What it controls:

* How much past data the model sees

---

### ⚖️ Effect

| Value | Result                           |
| ----- | -------------------------------- |
| Short | Miss long patterns               |
| Long  | More context but harder to train |

---

### 🧩 Analogy

Reading:

* 1 sentence → limited understanding
* Full paragraph → better context

---

## 5️⃣ Batch Size

```id="hp5"
batch_size = 16
```

### 🧠 What it controls:

* Number of samples processed together

---

### ⚖️ Effect

| Value | Result                       |
| ----- | ---------------------------- |
| Small | Noisy but flexible           |
| Large | Stable but needs more memory |

---

## 6️⃣ Number of Epochs

```id="hp6"
epochs = 200
```

### 🧠 What it controls:

* How many times model sees data

---

### ⚖️ Effect

| Value    | Result       |
| -------- | ------------ |
| Too few  | Underfitting |
| Too many | Overfitting  |

---

## 🔄 How to Tune Hyperparameters

---

## ✅ Method 1: Trial and Error (Most Common)

* Try different values
* Observe results
* Choose best

---

## ✅ Method 2: Grid Search

Try combinations:

```id="hp7"
learning_rate = [0.1, 0.01, 0.001]
hidden_size = [8, 16, 32]
```

👉 Test all combinations

---

## ✅ Method 3: Random Search

* Pick random values
* Faster than grid search

---

## 🧠 What to Watch During Tuning

### 📉 Loss Curve

* Should decrease smoothly

---

### 📊 Accuracy / Prediction Quality

* Should improve over time

---

### ⚠️ Overfitting Signs

* Training loss ↓
* Test loss ↑

---

## 🧩 Practical Example

Change this:

```python id="hp8"
model = nn.RNN(input_size=1, hidden_size=10)
```

To:

```python id="hp9"
model = nn.RNN(input_size=1, hidden_size=32)
```

👉 You may see:

* Better performance
* But slower training

---

## 🧠 Simple Strategy (Very Useful)

Start with:

* Learning rate = 0.01
* Hidden size = 8–16
* Epochs = 100–200

Then:

* Adjust step-by-step

---

## ⚠️ Common Mistakes

### ❌ Changing too many things at once

👉 You won’t know what worked

---

### ❌ Ignoring validation data

👉 Model may overfit

---

### ❌ Using very large models early

👉 Waste of time and resources

---

## 📌 Key Insight

> Hyperparameters control how well your model learns, not what it learns.

---

## 📌 One-Line Summary

> Hyperparameter tuning is the process of adjusting model settings to achieve the best performance.

---

## 🎯 What You Should Be Able to Do

After this chapter:

* Identify key hyperparameters
* Understand their effects
* Tune models step-by-step
