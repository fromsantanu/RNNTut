# 📘 Chapter 12.2: Common Mistakes

---

## 🧠 Why This Chapter Matters

Most problems in RNN projects don’t come from theory—they come from **small practical mistakes**.

👉 Fixing these early saves a lot of time.

---

# 🔴 1. Ignoring Input Shape (Most Common)

---

## ❌ Mistake

Wrong input format to RNN

---

## ✅ Correct Format

```id="cm1"
(batch_size, sequence_length, input_size)
```

---

## 🧠 Why It Matters

RNN reads data **step-by-step**
If shape is wrong → model cannot understand sequence

---

## 🧩 Tip

Always print shape:

```python id="cm2"
print(X.shape)
```

---

# 🔴 2. Not Using Last Time Step Properly

---

## ❌ Mistake

Using full sequence output incorrectly

---

## ✅ Correct Approach

```python id="cm3"
out = out[:, -1, :]
```

---

## 🧠 Why It Matters

* Many tasks need **final understanding**
* Last step contains full context

---

# 🔴 3. Wrong Learning Rate

---

## ❌ Mistake

* Too high → unstable
* Too low → very slow

---

## 🧠 Solution

Start with:

```id="cm4"
lr = 0.01
```

Then adjust gradually

---

# 🔴 4. Ignoring Vanishing/Exploding Gradient

---

## ❌ Mistake

Not handling gradient problems

---

## 🧠 Symptoms

* Model not learning
* Loss not changing
* Sudden large loss

---

## ✅ Solution

* Use LSTM/GRU
* Apply gradient clipping

---

# 🔴 5. Using RNN for Long Sequences

---

## ❌ Mistake

Using basic RNN for long text or time-series

---

## 🧠 Result

* Poor performance
* Forgetting early data

---

## ✅ Solution

Use:

* LSTM
* GRU

---

# 🔴 6. Overfitting the Model

---

## ❌ Mistake

* Model memorizes training data

---

## 🧠 Symptoms

* Training accuracy high
* Test accuracy low

---

## ✅ Solution

* Dropout
* Early stopping
* More data

---

# 🔴 7. Not Normalizing Data

---

## ❌ Mistake

Feeding raw values directly

---

## 🧠 Problem

* Large values → unstable training

---

## ✅ Solution

Normalize data:

```id="cm5"
X = (X - mean) / std
```

---

# 🔴 8. Too Complex Model Too Early

---

## ❌ Mistake

Using large models without understanding basics

---

## 🧠 Result

* Hard to debug
* Slow training

---

## ✅ Solution

Start simple → then scale

---

# 🔴 9. Ignoring Validation Data

---

## ❌ Mistake

Only checking training loss

---

## 🧠 Problem

* Cannot detect overfitting

---

## ✅ Solution

Use:

* Train set
* Validation set

---

# 🔴 10. Copy-Paste Without Understanding

---

## ❌ Mistake

Running code without knowing what it does

---

## 🧠 Result

* No real learning
* Cannot fix errors

---

## ✅ Solution

* Understand each line
* Modify and experiment

---

# 🔴 11. Not Using Enough Data

---

## ❌ Mistake

Training on very small dataset

---

## 🧠 Problem

* Model cannot learn patterns

---

## ✅ Solution

* Increase dataset
* Use augmentation

---

# 🔴 12. Expecting Perfect Predictions

---

## ❌ Mistake

Expecting exact output

---

## 🧠 Reality

* Models learn patterns
* Not exact rules

---

## ✅ Solution

* Focus on trends
* Evaluate reasonably

---

# 🧠 Summary of Mistakes

| Mistake              | Solution           |
| -------------------- | ------------------ |
| Wrong input shape    | Use correct format |
| Learning rate issues | Tune gradually     |
| Gradient problems    | Use LSTM/GRU       |
| Overfitting          | Regularization     |
| Complex models early | Start simple       |
